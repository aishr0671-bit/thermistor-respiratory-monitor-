01 · Motivation

Why We Built This

Respiratory rate is one of the most critical vital signs — yet it is the most overlooked in low-resource clinical settings. Apnea, the temporary cessation of breathing, can be fatal if undetected for even a few seconds in vulnerable patients such as neonates or post-operative adults.

We designed this monitor to be the simplest possible working system: no specialized ICs, no op-amps, no complex firmware — just a thermistor, a resistor, and an Arduino Nano. The goal was to demonstrate that a clinically meaningful device can be built from components available at any electronics shop.

Low-Resource Design

Built from off-the-shelf components. No calibration, no soldering, no specialized equipment required.

Critical Apnea Alert

LED and active buzzer trigger immediately when breathing stops — fast enough to matter in real clinical scenarios.

Thermistor Sensing

Exhaled breath warms the thermistor, lowering its resistance and raising the A0 voltage. Each breath = one complete cycle.

Working Principle

An NTC (Negative Temperature Coefficient) 10K thermistor is placed near the patient's mouth or nose, inside a mask. When the patient exhales, warm air heats the thermistor — its resistance drops — and the voltage at pin A0 rises. During inhalation, the thermistor cools and the voltage falls. The Arduino continuously samples A0, maintains a rolling baseline, and compares each reading to detect a breath event. If no breath is detected within the timeout window, the apnea alarm activates on D3 (LED solid) and D5 (active buzzer).

Critical Notes — Read Before Powering
Do NOT connect anything to the Vin pin — it is for 7–12V input only and must be left unconnected in this build. Use regulated 5V to the 5V pin only. All GND points must be tied to a common ground. If using a passive buzzer, a transistor driver circuit is required — use an active buzzer for direct connection. Place the thermistor close to the mouth or inside a mask for reliable breath detection.

03 · Software Design & Implementation

Firmware Logic

The firmware runs on the Arduino IDE (C/C++) and implements a continuous sampling loop. The ADC reads the thermistor voltage on pin A0 every ~100 ms. A slow exponential moving average tracks the patient's resting baseline. Each new sample is compared to this baseline — a deviation beyond the threshold is counted as a breath event. If no event occurs within the apnea timeout window, the alarm fires on D3 (LED solid) and D5 (active buzzer).

Loop · Continuous

ADC Sampling

Reads analog pin A0 every ~100 ms. Maps thermistor voltage to a 0–1023 integer value for processing.

Logic · Per sample

Baseline Tracking

Exponential moving average (0.98 / 0.02 weighting) tracks the resting temperature baseline across time.

Detect · Per cycle

Breath Detection

A breath is counted when |ADC − baseline| exceeds the threshold. Each detection resets the apnea timer.

Output · D3

LED Flash

D3 pulses HIGH briefly on each breath. Stays solid HIGH during apnea as an additional visual alert.

Alarm · D5

Active Buzzer

D5 goes HIGH during apnea, directly driving the active buzzer without any additional driver circuit.

Serial · 9600

Debug Output

Real-time ADC values and breath count printed to Serial Monitor at 9600 baud for calibration and testing.

Core Algorithm

// ---- Pin Definitions ----
const int THERM_PIN = A0;
const int LED_PIN = 3;
const int BUZ_PIN = 5;
const int THRESHOLD = 15; // ADC deviation to count as breath
const long APNEA_MS = 10000; // 10 second apnea timeout

float baseline = 512.0;
unsigned long lastBreath = 0;

void loop() {
  int val = analogRead(THERM_PIN);
  baseline = 0.98 * baseline + 0.02 * val;

  if (abs(val - baseline) > THRESHOLD) {
    digitalWrite(LED_PIN, HIGH);
    lastBreath = millis();
    delay(80);
    digitalWrite(LED_PIN, LOW);
  }

  if (millis() - lastBreath > APNEA_MS) {
    digitalWrite(LED_PIN, HIGH); // solid on = apnea
    digitalWrite(BUZ_PIN, HIGH); // active buzzer ON
  } else {
    digitalWrite(BUZ_PIN, LOW);
  }
  delay(100);
}
Note on Active vs Passive Buzzer — Because we use an active buzzer, it only needs a HIGH/LOW digital signal on D5 to sound. There is no need for tone() or a transistor driver. If a passive buzzer is substituted, a transistor (e.g. 2N2222) driver stage and a PWM tone() call are required.
04 · Conclusion

Results & Reflections

The device successfully detected breathing cycles in bench testing. The LED flashed reliably on each simulated breath, and the buzzer activated after the apnea timeout. Thermistor placement inside a mask gave the clearest signal. The optional 1 μF capacitor visibly reduced ADC jitter on a USB-powered supply.

Strengths

  • Entire circuit fits on one breadboard — minimal components
  • No op-amp or signal conditioning needed
  • Active buzzer connects directly to D5 — no driver required
  • Self-adapting baseline handles different patient temperatures
  • USB powered — no battery management required
  • Serial monitor output aids calibration and debugging

Future Work

  • Add OLED display to show live breaths-per-minute
  • Improve detection with adaptive peak-finding algorithm
  • Add op-amp amplifier stage for weaker thermistor signals
  • Log breath data to SD card for clinical record-keeping
  • Replace USB with battery + 5V regulator for portability
  • Validate against a reference spirometer or capnograph

This project was conducted in adherence to the IEEE Code of Ethics. The thermistor mask interface is non-invasive and poses no patient risk in its current form.
